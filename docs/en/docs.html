📡 SDR Receiver

Receive live ADS-B directly from aircraft — no internet needed

🔌

Hardware

RTL-SDR Blog V3 or any RTL2832U dongle (~$25). Included antenna works; dedicated 1090 MHz antenna extends range to 300+ km.

🐍

Software

Python 3.8+, pyModeS, Flask. Run sdr_receiver.py and connect from the web app.

pip install pyModeS flask flask-cors
python3 sdr_receiver.py --lat 48.86 --lon 2.35
# Then tap 📡 in the web app → Connect

Three backends supported: rtl_adsb (direct), dump1090 (network), SBS/BaseStation (CSV). Raspberry Pi systemd service file included for permanent deployment.

Frequently Asked Questions

Quick answers to common questions

Why don't I see any planes?
The OpenSky API has rate limits (~100 requests/day). Wait a few minutes, or press D for Demo Mode. Remote or ocean locations naturally have fewer flights. The app auto-switches to Demo Mode after 2 failed API calls.
Are these real planes?
Yes! When connected to OpenSky or SDR, these are real aircraft with real GPS positions, updated every 10–12 seconds. In Demo Mode (gold badge visible), flights are simulated.
Does it work offline?
The app itself loads instantly with no internet. Live data requires a connection (or an SDR dongle). Without internet, Demo Mode activates automatically.
Which browsers are supported?
Chrome, Edge, Safari, and Firefox all work. Web Bluetooth (for micro:bit) requires Chrome or Edge. All features work on mobile.
Can I use this in my classroom?
Absolutely! Lesson ideas: geography (filter by country, discuss flight routes), math (speed × time = distance), science (atmospheric layers, contrails), history (Heritage Mode), engineering (SDR signals). The five modes are a natural lesson progression.
Why a single HTML file?
Simplicity. Teachers can email it, students open it from a USB stick, there's nothing to install. Zero dependencies, zero supply-chain risk, instant loading.
Does it track my location?
Your location is used only to calculate which planes to show and for sunrise/sunset timing. Nothing is stored, tracked, or sent to third parties beyond the OpenSky API query.
Can I use both SDR and micro:bit together?
Yes! The SDR provides data, the web app displays it, and the micro:bit gives you a physical companion. All three work simultaneously.

Technical Details

Under the hood

Rendering

Canvas 2D at native DPI. 9 procedural aircraft shapes. 60fps animation loop.

Audio

Web Audio API. Oscillator-synthesized SFX. No external files.

Motion

DeviceOrientation API for gyroscope parallax. Pointer drag fallback.

Bluetooth

Web Bluetooth API for micro:bit UART service.

Astronomy

Solar declination algorithm for sunrise/sunset/twilight.

APIs

OpenSky Network (flights), Nominatim (geocoding), Google Fonts.
